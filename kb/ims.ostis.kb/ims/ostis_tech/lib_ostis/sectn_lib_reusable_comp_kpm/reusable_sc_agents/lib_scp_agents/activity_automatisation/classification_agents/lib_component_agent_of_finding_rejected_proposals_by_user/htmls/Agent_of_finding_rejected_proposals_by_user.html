Задачей агента поиска всех предложений, отклоненных конкретным пользователем является поиск отклоненных пользователем предложений. На вход данному агенту подается пользователь. Для данного пользователя будут найдены все предложения, которые он отклонил. При необходимости ограничения области поиска агента в качестве второго аргумента запроса подается контекст действия, в рамках которого будет осуществляться поиск. Контекст действия является структурой. Данный агент инициируется при условии появления в памяти вопросной конструкции, соответствующей действию. поиск всех отклоненных конкретным пользователем предложений. Возможные результаты работы агента:

  • Если на вход агенту пришел не знак пользователя, то генерируется сообщение об ошибке (Wrong first parameter! It must be user sign(element of ui_user)).
  • Если на вход агенту был подан второй аргумент, который не является знаком структуры, то генерируется сообщение об ошибке (Wrong second parameter! It must be a structure).
  • Если в результате поиска отклоненные пользователем предложения не были найдены, то генерируется сообщение "The search hasn't given any results".
  • Если в результате поиска отклоненные пользователем предложения были найдены, то выводятся все найденные предложения. Пример результата работы агента представлен на рисунке. Класс действия (смотреть рисунок) должен являться ключевым sc-элементом Раздел. Предметная область действий разработчиков sc-моделей баз знаний, соответствующих методике, предполагающей взаимодействие всех разработчиков с семантической памятью и наличием иерархии администраторов. Если класс действия не является ключевым sc-элементом данного раздела, то данный класс действия не будет добавлен в результат работы агента.
