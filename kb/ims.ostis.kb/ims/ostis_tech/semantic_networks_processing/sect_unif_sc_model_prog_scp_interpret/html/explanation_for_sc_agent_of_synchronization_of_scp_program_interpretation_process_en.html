The task of the sc-agent of synchronization of scp-program interpretation process is supporting of transitions between scp-operators within one scp-program. This sc-agent is activated when adding some scp-operators in the set of executed scp-operator. Transition to sc-arcis carried further, belonging to the relation of the goto scp-operator* (or more private relations, in case, if scp-operator was added in the set of successfully executed scp-operators or unsuccessfully executed scp-operators). Thus next scp-operator becomes active scp-operator only if all scp-operators, which is related entering sc-arcs, inherings to the relation goto scp-oprator* (or more private relations) became set members executed_scp-operators (or, respectively, its subsets). Otherwise there is a waiting of completion of required scp-operators.
